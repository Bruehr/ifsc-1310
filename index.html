“Thoughts on Web Accessibility”

Advancements have in Web Accesibility have made interacting and contributing to the web easier for groups of people who would have had trouble otherwise. Web Accessibility has helped people with disabilites, such a auditory and visual problems, have a better experience on the web. It also helps people without disabilites who might be using their mobile phone or smart TV to access the web. Whether your disabled or not Web Accesibility can make a better experience for all on the web.

What are some things you can to do to make the Web more Accessible
  • You can add images with Alt Text
  • You can allow users enlarged font sizes
  • Contrast Sensitivity
  • You can add Keyboard Navigation
  • Make Video and Multimedia Accessible
  • Descriptive URLs
  • ARIA rules
  • Avoid Using Placeholder Text in Forms
  • Minimize Use of Tables

Progressive Enhancement Writing Assignment

Progressive enhancement is a website building strategy that focuses on building from the content first and then adding all the bells and whistles after you have built a good base. The first layer you will build is the HTML layer which is the structures of the page. A well structure HTML document will produce a page with all the content a user would need to see its bare bone nature. This is important for older browsers whom might not support all the bells and whistles. The next layer is to add the style to the page with CSS. With this you can enhance your site with different color options, different fonts, and oven did layouts for various types of content. The third layer for progressive enhancement is JavaScript. JavaScript can be used to affect the behavior of your site; you can use it to create dynamic content such as an image carousals or interactive elements for the user. HTML, CSS, and JavaScript build upon each other progressively to create the webpages you see today.
